# 데이터 명세서 — 대한민국 개별공시지가 시계열 (1990–2026)

전국 모든 필지의 **개별공시지가(원/㎡)**를 제도 시작 시점인 1990년부터 2026년까지, 공신력 있는 정부 출처만으로 수집·정규화한 시계열 데이터셋입니다.

> **한눈에 보기** → [요약 대시보드](index.html) · [시도별 지도](map.html)

---

## 1. 개요

| 항목 | 값 |
|---|---|
| 수록 기간 | **1990 ~ 2026년** (37개 연도) |
| 총 레코드 | 약 **11.5억 행** (1,148,663,901) |
| 관측 단위 | 필지(PNU) × 연도 × 공시월 |
| 값 | 개별공시지가 (원/㎡) |
| 공간 범위 | 대한민국 전국 (17개 시도, 250개 시군구) |
| 저장 포맷 | Apache Parquet (zstd 압축), 약 7.2 GB |
| 좌표/도형 | **없음** — 속성(값) 테이블. 지도화는 PNU로 경계·지적도에 조인 (§7) |

**개별공시지가란?** 국토교통부장관이 정한 표준지공시지가를 기준으로 시장·군수·구청장이 산정·공시하는 **개별 토지의 단위면적당(㎡) 적정가격**입니다. 재산세·양도세 등 각종 조세와 부담금의 산정 기준으로 쓰이는 **공적 평가가격**이며, 시장 실거래가와는 다릅니다.

## 2. 스키마

각 행은 "특정 필지의 특정 공시 시점 지가" 하나를 나타냅니다.

| 컬럼 | 타입 | 설명 | 예시 |
|---|---|---|---|
| `pnu` | string(19) | 필지 고유번호 (§3 참조) | `1111010100100300021` |
| `stdr_year` | int16 | 공시 기준연도 | `2026` |
| `stdr_month` | int8 | 공시 기준월 · **1** = 정기공시(1월 1일) / **7** = 추가공시(7월 1일) | `1` |
| `price_per_m2` | int64 | 개별공시지가 (원/㎡) | `2137000` |
| `pblntf_de` | string | 공시일자 (YYYY-MM-DD) | `2026-04-30` |
| `std_land_yn` | bool | 표준지 여부 | `false` |
| `source` | string | 출처 (`vworld_api` / `seoul_opendata`) | `vworld_api` |
| `collected_at` | date | 수집일자 | `2026-07-17` |

**자연 키**: (`pnu`, `stdr_year`, `stdr_month`, `source`). 이 조합은 유일합니다 (중복키 0, §6).

## 3. PNU — 필지 고유번호 (19자리)

PNU(Parcel Number)는 모든 필지를 유일하게 식별하는 코드이며, 자릿수마다 의미가 있어 **행정구역 집계와 조인의 핵심 키**입니다.

```
1111010100 1 0030 0021
└────┬────┘ │ └┬─┘ └┬─┘
 법정동코드  │  본번  부번
  (10자리)  대장구분(1)
```

| 구간 | 자리 | 의미 |
|---|---|---|
| `pnu[0:2]` | 2 | **시도** (11 서울, 26 부산, 41 경기 …) |
| `pnu[0:5]` | 5 | **시군구** (11110 종로구 …) |
| `pnu[0:10]` | 10 | **읍면동·리** (법정동코드) |
| `pnu[10:11]` | 1 | 대장구분 (1 일반 / 2 산 …) |
| `pnu[11:15]` | 4 | 본번 |
| `pnu[15:19]` | 4 | 부번 |

> 활용: `substr(pnu,1,5)`로 시군구별 집계, `substr(pnu,1,2)`로 시도별 집계. 지적도의 `PNU` 필드와 조인하면 필지별 지도가 됩니다.

## 4. 시간 커버리지와 밀도

- **시작점 1990년**: 개별공시지가 제도의 최초 고시(1990-08-30)가 데이터의 시작입니다. 그 이전은 필지 단위 공적 지가가 제도적으로 존재하지 않습니다.
- **정기공시 (`stdr_month=1`)**: 매년 1월 1일을 기준일로 전 공시대상 필지에 대해 공시 (통상 4~5월 공시일).
- **추가공시 (`stdr_month=7`)**: 상반기(1.1~6.30)에 분할·합병·신규등록·지목변경 등이 발생한 필지에 한해 7월 1일 기준으로 추가 공시 (10월경).
- 따라서 제도상 **최대 시간 밀도 = 연 1회 전 필지 + 변경 필지 한정 연중 1회 추가**입니다. 이보다 촘촘한 필지 단위 공적 지가는 존재하지 않습니다.

## 5. 파일 레이아웃

```
data/parquet/individual/
├── stdr_year=1990/
│   ├── sgg=11110-vworld_api.parquet
│   ├── sgg=11110-seoul_opendata.parquet
│   └── … (시군구 × 출처별)
├── stdr_year=1991/
│   └── …
└── stdr_year=2026/
    └── …
```

- **파티셔닝**: `stdr_year`(연도) / `sgg`(시군구 5자리) / `source`(출처). 연도·지역 슬라이스 쿼리에 최적화.
- **출처 병존**: 서울은 두 출처(`seoul_opendata` 벌크 + `vworld_api`)가 파일명으로 구분되어 공존합니다. 값이 필요한 분석에서는 한 출처만 선택하거나 `source`로 필터하세요.
- Hive 파티셔닝이라 DuckDB/GDAL 등에서 `stdr_year`, `sgg`가 컬럼으로 자동 인식됩니다.

## 6. 데이터 품질

이 데이터셋은 다음 검증·정리를 거쳤습니다.

**① 다중 개정본 정리** — V-World API는 한 필지의 여러 개정 버전을 모두 반환합니다(공시 후 정정되면 옛 값과 새 값을 함께 반환, `lastUpdtDt`로 구분). 수집이 며칠에 걸쳐 돌면 개정 전·후 값이 함께 저장되므로, **필지별로 `lastUpdtDt`가 가장 최근인 값(=official)만** 남겼습니다.
- 완전중복(값 동일) 약 716만 행 제거
- 가격충돌(값 상이) 385개 필지는 API 재조회로 최신 확정값 반영
- 정리 후 (pnu·연도·월·출처) 중복키 **0**

**② 교차검증** — 서울 1990년 전수(43,896필지)를 서울시 공식 데이터(열린데이터광장)와 API로 대조 → 가격 불일치 **0**.

**③ 페이지네이션 무결성** — 대형 법정동의 페이지 분할 수집에서 누락 **0** (수집 행수 = 서버 신고 총건수).

**④ 행정구역 개편 대응** — 코드가 바뀐 지역의 과거 데이터를 폐지 코드까지 포함해 완전 수집. (강원 42→51, 전북 45→52, 전남·광주 통합 등)

**⑤ 알려진 특성 (오류 아님)**
- **2006년 필지 수가 낮음** (전국 약 27M vs 전후 29~30M): 원본 데이터 특성으로 확인. 공식 서울 데이터와 API가 독립적으로 동일하게 낮고, 227개 시군구 전역에서 균일하게 나타남 (특정 지역 급감 없음).

## 7. 지도화 방법 (도형 조인)

이 데이터엔 값만 있으므로, 지도로 그리려면 **도형에 키로 조인**합니다.

**지역(시도·시군구) 단위** — [docs/summary_data/](summary_data)의 집계 CSV(연도별 중위·평균·필지수)를 행정구역 경계에 `sgg_key`/시도명으로 조인. 별도 도형 없이 QGIS에서 바로 choropleth 가능.

**필지 단위** — 실제 필지 폴리곤(**연속지적도**)이 필요합니다.
1. V-World 공간정보 다운로드에서 연속지적도(시군구별 SHP, `PNU` 속성 포함) 확보
2. 필지 지가 테이블을 `pnu` = 지적도 `PNU`로 조인
3. `price_per_m2`로 색칠 → 필지별 지가 지도

> 전국 필지 폴리곤은 수십 GB이므로, 실무에서는 분석 대상 시군구만 지적도를 받아 조인합니다.

## 9. 데이터 얻기 / 재현

전체 원본(약 7.2GB)은 용량상 저장소에 포함되지 않습니다. 아래 중 하나로 확보합니다.

**A. 파이프라인 실행 (재현)** — V-World 개발키 발급 후 `.env`에 설정하고 순서대로 실행. 상세는 루트 [README.md](../README.md).

**B. 요약 데이터로 충분한 경우** — 지역 단위 시계열은 [docs/summary_data/](summary_data)에 포함되어 있어 바로 사용 가능합니다.
- `시도별_공시지가_1990-2026.csv`
- `시군구별_공시지가_1990-2026.csv`
- 컬럼: `키, 시도, 시군구, 연도, 중위지가, 평균지가, 필지수`

## 10. 사용 예시 (DuckDB)

```sql
-- 특정 필지의 전 기간 시계열
SELECT stdr_year, price_per_m2
FROM read_parquet('data/parquet/individual/**/*.parquet', hive_partitioning=true)
WHERE pnu = '1111010100200090004' AND stdr_month = 1
ORDER BY stdr_year;

-- 2026년 시군구별 중위 공시지가
SELECT substr(pnu,1,5) AS sgg,
       median(price_per_m2) AS med_price,
       count(*) AS parcels
FROM read_parquet('data/parquet/individual/stdr_year=2026/*-vworld_api.parquet')
WHERE stdr_month = 1 AND price_per_m2 > 0
GROUP BY 1 ORDER BY 2 DESC;
```

## 11. 유의사항 요약

- 공시지가는 **공적 평가가격**이며 실거래가·시가와 다릅니다.
- 1990년 이전 필지 단위 공적 지가는 존재하지 않습니다.
- 7월 추가공시는 변경 필지에만 존재하므로 패널이 완전 균형(balanced)은 아닙니다.
- 행정구역이 개편된 지역은 같은 땅이 연도별로 다른 코드의 PNU로 기록될 수 있습니다 (원천시도시군구 이력 기준).
- 재배포 시 출처(국토교통부·서울시·통계청) 표기를 권장합니다.
