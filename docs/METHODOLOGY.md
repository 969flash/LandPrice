# 구축 방법론 (METHODOLOGY)

목표: **대한민국 전체 필지의 개별공시지가를, 제도상 기록 가능한 최초 시점(1990년)부터 최대 밀도(연 1회 + 7.1 추가공시)로** 수집·정규화·저장한다.

## 1. 시간 범위와 밀도의 근거

- 필지 단위 공적 지가의 시작점은 **1990년** (개별토지가격 최초 고시 1990-08-30). 그 이전은 필지 단위 공개 데이터가 제도적으로 존재하지 않음.
- 제도상 최대 밀도 = **연 1회 (1월 1일 기준) + 상반기 변경 필지 한정 7월 1일 기준 추가공시**. 데이터의 `stdr_month` (1 또는 7)로 구분 수록.
- 월 단위 신호는 필지 단위로 존재하지 않음 → 선택적으로 지가변동률(시군구×용도지역 표본 지수)을 **별도 파생 레이어**로 결합 (원본과 혼합 금지).

## 2. 수집 전략 (기존 공개 데이터 최대 활용 원칙)

우선순위 순서:

| 순위 | 경로 | 커버리지 | 역할 |
|---|---|---|---|
| 1 | 서울 열린데이터광장 벌크 (OA-1180) | 서울 × 1990~2026 | 무인증 즉시 적재 + API 교차검증 기준 |
| 2 | V-World 데이터마켓 벌크 (dsId=6) | 전국 × 최근 ~4-5개년 | 최근 연도 고속 적재 (로그인 필요) |
| 3 | **V-World API 스윕** (`getIndvdLandPriceAttr`) | 전국 × 1990~현재(기대) | 나머지 전부 — 유일한 전 기간 자동화 경로 |
| 4 | data.go.kr 표준지 CSV + API | 전국 표준지 | 표준지 시계열 (별도 테이블) |

### API 스윕 설계

- 쿼리 키: **법정동코드 10자리** (code.go.kr 전체자료의 동·리 leaf 코드)를 `pnu` 접두어로 사용, `stdrYear` × `pageNo`(1000행/페이지) 순회.
- leaf 판정: 읍면동 자리(6~8째 자리) ≠ '000'인 코드 중, 리 자리가 '00'인 코드는 동일 8자리 접두어의 리(≠'00') 코드가 없을 때만 leaf.
- **폐지 코드 반드시 포함** (2026-07-15 실측 후 확정): 행정구역 개편 시 과거 데이터의 귀속 코드가 **지역·개편 시기마다 다름**을 확인.
  - 재키잉 완료된 사례: 서울 강북구(1995 신설 코드에 1990 데이터 있음), 강원 42→51(신코드에 1990 데이터), 세종(신코드에 1995 연기군 데이터)
  - **재키잉 안 된 사례: 전북(45→52, 2024) — 1990~2023 데이터가 폐지된 45xxx에 잔존. 전남(46)·광주(29) — 전남광주통합특별시(12xxx) 신설 후에도 과거~현재 데이터 전부가 폐지된 옛 코드에 잔존** (신코드 조회 시 0건)
  - 따라서 현존 코드만 스윕하면 위 지역이 통째로 누락됨 → 현존+폐지 전체(49,562 leaf)를 스윕. 신설 코드의 과거 연도 0건은 정상(데이터는 옛 코드에 있음).
  - **후속 과제**: 동일 필지가 연도에 따라 다른 코드 체계의 PNU로 나타나므로(예: 전주 완산구 45111→52111), 필지 시계열 결합용 신구 코드 매핑 테이블 구축 필요 (행정표준코드 연혁 기반; 시도 개칭은 접두어 치환, 구 신설·부활은 개별 매핑).
- 작업 단위: **(시군구 5자리, 연도)** — 완료 시 해당 파티션 Parquet 1개 기록 + SQLite 체크포인트. 중단 후 재시작 시 완료 단위 스킵.
- 속도: 기본 8 req/s (보수적). 지수 백오프 재시도 (5xx/타임아웃/네트워크). `resultCode != OK`는 유형별 처리 (INVALID_KEY 즉시 중단, 무데이터는 정상 스킵).
- 규모 추정: 공시 대상 약 3,560만 필지 × 37개 연도 ≈ **10~13억 행**, 최소 요청 수 ≈ 130만+ (페이지 수 기준) → 8 req/s 연속 가동 시 약 2~5일. 연도 부분집합 실행 옵션 제공.

## 3. 표준 스키마 (개별공시지가 사실 테이블)

모든 출처를 아래로 정규화. Parquet, zstd 압축.

| 컬럼 | 타입 | 설명 |
|---|---|---|
| `pnu` | string(19) | 필지 고유번호 = 법정동코드(10)+대장구분(1)+본번(4)+부번(4). 특수지는 19자리 초과 가능 |
| `stdr_year` | int16 | 공시 기준연도 (1990~) |
| `stdr_month` | int8 | 1 = 1.1 기준 정기공시, 7 = 7.1 기준 추가공시 |
| `price_per_m2` | int64 | 공시지가 (원/㎡) |
| `pblntf_de` | date (nullable) | 공시일자 (출처가 제공하는 경우만) |
| `std_land_yn` | bool (nullable) | 표준지 여부 |
| `source` | string | `seoul_opendata` / `vworld_api` / `vworld_bulk` |
| `collected_at` | date | 수집일 |

- 파생 컬럼 (쿼리 편의): `sido_cd` = pnu[:2], `sgg_cd` = pnu[:5] — 파티션 키로만 사용.
- **자연 키**: (`pnu`, `stdr_year`, `stdr_month`). 중복 출처 병합 시 우선순위: vworld_api = vworld_bulk > seoul_opendata (동률 시 최신 collected_at). 불일치 발견 시 검증 리포트에 기록.

## 4. 저장 구조

```
data/
├── raw/                          # 원본 그대로 (재현성 — 삭제 금지)
│   ├── seoul/공시지가_YYYY년.zip
│   ├── bjd/법정동코드.zip
│   └── vworld_bulk/…
├── parquet/
│   ├── individual/               # 개별공시지가 사실 테이블
│   │   └── stdr_year=YYYY/sgg=XXXXX.parquet
│   └── standard/                 # 표준지 (별도 테이블)
└── db/checkpoints.sqlite         # 스윕 진행 상태
```

- 파티셔닝: `stdr_year` / `sgg`(시군구 5자리) — 연도·지역 슬라이스 쿼리 최적화, 파일 수 약 9천여 개로 관리 가능.
- 전체 예상 크기: 10~13억 행 × 압축 후 행당 ~15-25바이트 ≈ **20~35GB** (Parquet zstd). 원본 보존 포함 여유 100GB 권장.
- 조회: DuckDB로 파티션 전체를 단일 뷰로 취급 (`read_parquet('data/parquet/individual/**/*.parquet', hive_partitioning=true)`).

### 서울 처리 방침 (2026-07-15)

속도 확보를 위해 서울 25개 자치구 × 전 연도는 **열린데이터광장 벌크로 적재하고 API 스윕에서 제외** (체크포인트에 완료 표기). 이로 인해 서울은 ① 7.1 추가공시분, ② API에만 있는 일반 필지 꼬리(~0.1%)가 일시 누락 → **전국 스윕 완료 후 서울 API 보강 패스**로 채움 (체크포인트에서 서울 단위 삭제 후 재실행).

## 5. 품질 검증 (QA)

1. **교차검증**: 서울 임의 표본 (연도×자치구별 n필지)에 대해 서울 벌크 값 vs API 값 일치율 측정 (목표 100%, 불일치는 전수 기록).
2. **완전성**: 연도별 행 수를 국토교통부 공표 공시 대상 필지 수와 대조 (예: 2025년 약 3,559만). 시군구별 행 수 0인 파티션 자동 플래그.
3. **분포 검증**: 연도별 가격 분포 (음수/0/극단값), PNU 형식 (19자리, 법정동코드 존재 여부) 검사.
4. **재현성**: 원본(raw) 보존 + 모든 변환은 스크립트로만 수행 + 수집일 기록.

## 5.1 다중 개정본 처리 (2026-07 추가)

V-World API는 한 필지의 **여러 개정 버전을 모두 반환**한다. 공시 후 정정이 일어나면 옛 값과 새 값이 함께 오며, `stdrMt` 형식('01' 제로패딩 vs '1')과 `lastUpdtDt`(최종수정일)로 구분된다. 수집이 며칠에 걸쳐 돌면 개정 전·후 값이 모두 저장되어 (pnu, 연도, 월) 키에 행이 2개 이상 생긴다.

- **원칙**: `lastUpdtDt`가 가장 큰(최근 수정된) 레코드가 official 값.
- **파이프라인 반영**: `storage.dedup_latest()`가 수집 시 필지별 최신 개정본만 남긴다 (`scripts/04` 적용).
- **기존 데이터 소급 정리**: `scripts/09_conflict_census.py`(현황 집계) → `scripts/10_dedup_vworld.py`(전 파티션 중복 제거 + 가격충돌 필지 재조회 확정).
- **실측 규모(2026-07)**: 완전중복(값 동일) 약 716만 행 제거, 가격충돌 385개 필지 재조회 확정. 정리 후 (pnu,연도,월,source) 중복키 0.

## 6. 알려진 한계 (명시적)

- **2006년 필지 수가 이웃 연도보다 낮음**(전국 약 27M vs 29~30M, 서울 약 63만 vs 93만): 수집 오류 아님. 공식 서울 열린데이터광장과 V-World API가 독립적으로 동일하게 낮고, 비서울 227개 시군구 전부 균일하게 ~90% 수준(절반 이하 급감 시군구 0개)이라 원본 데이터의 특성으로 확인됨.

- **서울 벌크 파일은 1월 기준분만 수록** (1990·1991·2025 실측 — 기준년월 7월 행 0건). 7.1 추가공시분은 V-World API로만 확보 가능 → 서울도 7월분 보강 스윕 필요 (`stdrMt` 구분 수집).

- 1990년 이전 필지별 공적 지가는 존재하지 않음 (4원화 체계, 비공개).
- 7.1 추가공시는 변경 필지에만 존재 — 패널이 완전 균형(balanced)이 아님.
- 행정구역 개편으로 원천 DB가 **현행 PNU로 재키잉**되어 있어, 과거 시점의 당시 행정구역 복원이 필요하면 별도 연혁 코드 매핑 필요 (원천시도시군구코드 컬럼 활용).
- 공시지가는 감정평가 기반 공적 가격이며 시장 실거래가와 다름 (실거래가는 국토교통부 실거래가 공개시스템 — 본 프로젝트 범위 외).
- API의 1990년대 소급 범위는 키 발급 후 실측 확인 필요 (probe 스크립트 제공). 미달 시 대안: 지자체별 정보공개청구 또는 국토교통부 국가공간정보센터 데이터 제공 신청.
